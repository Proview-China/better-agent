# CMP Five-Agent Configuration And Controlled Reintegration Outline

状态：下一阶段执行总纲。

更新时间：2026-03-25

## 这份文档回答什么

上一阶段我们已经完成了两件事：

1. `CMP` 非五-agent 底座收口
2. 五-agent runtime 首轮代码骨架接入主链

下一阶段不再继续扩公共底座，也不直接扑到“无限联调”。

下一阶段真正要做的是：

- 先把五个 agent 的配置面做对
- 再把 `TAP` 的能力面以角色差异化方式接进来
- 再做受控联调

一句白话：

- 现在不是先继续写更多 runtime 壳子
- 而是要把五个 agent 真正“调成五个不同的人”

## 当前唯一目标

在不破坏已经跑通的：

- `core_agent -> rax.cmp -> cmp-runtime -> five-agent runtime`

这条链的前提下，完成下面三件事：

1. 五套角色 prompt / profile / capability contract
2. `TAP -> CMP` 的基础能力接线
3. 受控链路联调与回卷恢复策略落地

## 当前阶段的执行原则

### 1. 主动主链优先

第一成功标准仍然是：

- 主动主链稳定

所以优先顺序固定为：

1. `ICMA`
2. `Iterator`
3. `Checker`
4. `DBAgent`
5. `Dispatcher`

### 2. 配置先于联调

下一波不先做“满天飞的联调”。

下一波正确顺序是：

1. 先配五套角色行为
2. 再接基础能力
3. 再做联调

### 3. `TAP` 接线先基础后扩展

下一波第一优先接的不是花哨工具，而是：

- git
- db
- mq

这三类基础能力。

之后再补：

- 工具能力

### 4. 父子协作仍然受控

当前阶段不允许把父子协作做成：

- 常态持续同步
- 任意平级互通

当前阶段默认规则仍然是：

- 子链独立推进
- 漂移或不足时，子链向父侧受控再申请
- 同父平级交换走：
  - 父 `DBAgent` 处理
  - 父 `core_agent` 显式批准

### 5. 观测先服务联调

当前阶段观测优先级是：

1. 每角色阶段
2. 包流向
3. 链路卡点

不先做：

- 成本治理
- 复杂经营看板

## 五个 agent 下一阶段的具体目标

### 1. `ICMA`

下一阶段重点：

- 固定系统挂片模板
- 固定输入切块策略
- 固定父级种子进入后的二次装配纪律

必须守住：

- 只挂片
- 不改根 `system`

### 2. `Iterator`

下一阶段重点：

- 固定 `commit` 级推进策略
- 固定与 `TAP` 的 git 基础能力契约
- 固定 review ref 的稳定命名和使用纪律

### 3. `Checker`

下一阶段重点：

- 固定“证据重整 / 历史检查 / 子任务精裁”的真正行为
- 固定父 `Checker` 在父 `DBAgent` 主审下的辅助职责
- 固定“子链精裁”和“父链辅助重整”的提示词边界

### 4. `DBAgent`

下一阶段重点：

- 固定主包 / 时间线附包 / task snapshot 的真实生成策略
- 固定子节点再干预请求的入口逻辑
- 固定父侧主审和再出包的行为

### 5. `Dispatcher`

下一阶段重点：

- 固定包流向策略
- 固定 `child_seed` 只进子 `ICMA`
- 固定 peer exchange 需要父主 agent 显式批准的链路
- 固定 passive return 的回送规则

## 下一阶段不要做错的事

- 不要再把注意力拉回 `CMP` 非五-agent 底座。
- 不要把五个 agent 的 prompt 合并成一套大 prompt。
- 不要先接大量外部工具再补 `git/db/mq` 基础能力。
- 不要把父 `DBAgent` 主审理解成父 `Checker` 失去意义。
- 不要把子节点再干预做成持续半同步。
- 不要把人工 override 当常规运营手段。
- 不要先做大而全观测面，忽略角色阶段和包流向。

## 下一阶段的完成定义

只有同时满足下面这些条件，才算这一阶段真正完成：

1. 五套角色 prompt / profile / capability contract 全部成型
2. `TAP` 已经把 `git/db/mq` 三类基础能力按角色差异化接进 `CMP`
3. 子节点再干预链可以受控跑通
4. 同父平级交换的“父 `DBAgent` 处理 + 父主 agent 显式批准”链可以跑通
5. 失败恢复已经按“同链路回卷恢复”进入真实联调
6. `rax.cmp` 已经能看见角色阶段和包流向层面的关键状态
