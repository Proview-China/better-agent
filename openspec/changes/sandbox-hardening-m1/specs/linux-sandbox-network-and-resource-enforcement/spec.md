## 新增需求

### 需求:Linux 禁网语义必须尽量真实实现
在 Linux 上，当 `network_access=false` 时，core 必须优先使用系统原生能力形成真实禁网语义。M1 的主路线应优先尝试 network namespace；若当前运行环境无法启用该能力，必须显式返回不支持，而不是仅通过命令黑名单冒充禁网。

#### 场景:network_access false 且 netns 可用
- **当** 请求关闭网络访问且当前环境支持 network namespace
- **那么** 执行器必须在 Linux 上形成真实禁网环境
- **那么** 审计信息必须说明网络隔离已启用

#### 场景:network_access false 但 netns 不可用
- **当** 请求关闭网络访问但当前环境不支持 network namespace
- **那么** 结果必须显式返回不支持或阻断
- **那么** UI 不得把该执行视为已真实禁网

### 需求:Linux baseline 资源限制必须区分生效与未生效
在 Linux 上，baseline 资源限制必须明确区分“已实际生效”的限制与“仅请求但当前未启用”的限制。至少需要覆盖 CPU、内存、文件大小、文件描述符和进程数这类限制的状态回显。

#### 场景:资源限制请求被应用
- **当** 请求提供可被当前环境支持的资源限制
- **那么** 执行结果中必须标明这些限制已生效
- **那么** 审计信息中必须回显实际限制值

#### 场景:资源限制请求当前未支持
- **当** 请求提供的资源限制在当前 Linux 环境下无法生效
- **那么** 执行结果中必须明确标识未启用或不支持
- **那么** 不得把该请求伪装成已受限制执行
