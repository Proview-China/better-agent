## 新增需求

### 需求:Linux 文件系统隔离必须使用系统原生能力优先实现
在 Linux 上，`shell` 与 `code` 沙盒必须优先使用系统原生能力实现文件系统限制，不依赖额外外部工具。M1 的主路线应以 Landlock 为优先实现，以限制可读路径、可写路径、工作目录与产物目录。

#### 场景:受限工作目录执行
- **当** 执行请求仅允许访问指定工作目录与产物目录
- **那么** core 必须在 Linux 上对文件系统访问施加真实限制
- **那么** 越权读写不得仅依赖逻辑层检查

### 需求:文件系统隔离结果必须可审计
Linux 文件系统隔离执行后，core 必须把实际启用的隔离方式、允许路径和回退状态写入执行结果或审计证据，供 UI 直接展示。

#### 场景:Landlock 成功启用
- **当** Linux 执行器成功启用了 Landlock
- **那么** 结果中必须记录文件系统隔离已启用
- **那么** 审计信息必须说明生效的读写路径边界
